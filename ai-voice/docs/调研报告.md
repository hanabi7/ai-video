### 3. 选型建议

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| **中文短剧配音** | CosyVoice (自托管) / MiniMax (API) | 中文自然度最高 |
| **快速原型/MVP** | 百度 TTS API | 免费额度足，接入简单 |
| **方言需求** | 讯飞 TTS | 支持16种中国方言 |
| **语音克隆** | CosyVoice3 / F5-TTS | 3秒样本即可克隆 |
| **离线/嵌入式** | Piper / CosyVoice | 轻量化，无需联网 |
| **多语言** | Coqui TTS / Bark | 支持100+语言 |

## 二、推荐解决方案：AI-Voice 配音系统

### 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                      AI-Voice 配音系统                        │
├─────────────────────────────────────────────────────────────┤
│  前端 (React)  │  CLI 工具      │  API 后端 (Node.js/Python)   │
│  ───────────  │  ───────       │  ─────────────────────────   │
│  • 文字转语音  │  • 快速合成    │  • 路由控制                   │
│  • 语音克隆   │  • 批量处理    │  • 任务队列                   │
│  • 音色管理   │  • 配置管理    │  • 多引擎适配层               │
│  • 项目导出   │                │                              │
├─────────────────────────────────────────────────────────────┤
│                    TTS 引擎层                                │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │CosyVoice │ │ 百度 TTS │ │ MiniMax  │ │ 讯飞 TTS │        │
│  │ (本地)   │ │  (API)   │ │  (API)   │ │  (API)   │        │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘        │
└─────────────────────────────────────────────────────────────┘
```

### 技术选型

| 组件 | 技术栈 | 说明 |
|------|--------|------|
| 后端 | Node.js + Express | 轻量快速，生态丰富 |
| 本地TTS | Python + CosyVoice | 中文效果最佳，支持GPU加速 |
| 前端 | React + TypeScript + Ant Design | 成熟稳定，组件丰富 |
| CLI | Node.js + Commander.js | 跨平台，开发高效 |
| 队列 | Bull + Redis | 处理并发任务 |
| 存储 | SQLite / PostgreSQL | 轻量或生产级 |

### 核心功能

1. **多引擎支持**：CosyVoice(本地) + 百度/讯飞/MiniMax(API)
2. **语音合成**：文字转语音，支持SSML，情感控制
3. **语音克隆**：3秒样本克隆音色
4. **音色管理**：预设音色库，支持自定义上传
5. **项目管理**：分角色、分场景管理配音任务
6. **批量处理**：支持长文本自动分段合成
